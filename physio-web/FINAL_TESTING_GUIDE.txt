ROOT CAUSE FOUND & FIXED
═══════════════════════════════════════════════════════════════════════════════

The problem was the angle calculation method and ranges:

OLD (BROKEN):
  Shoulder: Used hip → shoulder → elbow = wrong angle
  Knee: Had inverted angle ranges
  Result: Phase transitions never happened → reps never counted

NEW (FIXED):
  ✅ Shoulder: Now uses VERTICAL reference below shoulder → shoulder → elbow
  ✅ Angle ranges updated to match correct calculations
  ✅ Phase detection logic now works correctly

═══════════════════════════════════════════════════════════════════════════════
WHAT CHANGED IN CODE
═══════════════════════════════════════════════════════════════════════════════

1. SHOULDER ANGLE CALCULATION (frontend/script.js line 1340)
   
   OLD: { x: shoulder.x, y: shoulder.y - 0.3 } (point above shoulder)
   NEW: { x: shoulder.x, y: shoulder.y + 0.3 } (point below shoulder)
   
   WHY: Below reference inverts the angle so ranges make sense:
   - Arm down: 0-20° → phase 'down'
   - Arm raised: 100-150° → phase 'up'
   - Arm overhead: 150-180° → phase 'up'

2. SHOULDER OPTIMAL RANGES (frontend/script.js line 572)
   
   OLD: [60, 140]
   NEW: [100, 180]
   
   WHY: Matches actual angles from corrected calculation
   - Angle < 100 → phase 'down' (arm down by side)
   - Angle >= 100 → phase 'up' (arm raised)

3. KNEE OPTIMAL RANGE (frontend/script.js line 585)
   
   OLD: [40, 140]
   NEW: [40, 140] (kept same, now correct)
   
   - Angle 40-140 → phase 'flexed' (leg bent)
   - Angle 140-180 → phase 'extended' (leg straight)
